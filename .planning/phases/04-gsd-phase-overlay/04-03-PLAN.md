  <action>
    **1. Proportional Phase Navigator Bar** — `draw_gsd_phase_navigator()` in draw_panels.rs:

    Guard: only render when `state.color_mode == ColorMode::GsdPhase`.
    If no gsd_phase_report: show "No .planning/ directory found" as small weak RichText, return.
    If gsd_phase_running: show "Scanning GSD phases..." as small weak RichText, return.

    Render as a **proportional horizontal bar** — NOT uniform buttons:
    - Total width = available UI width (ui.available_width())
    - Each phase segment width = (phase.files.len() as f32 / total_files as f32) * total_width
    - Minimum segment width = 40px (so small phases remain clickable)
    - Fill color = gsd_phase_color(phase.status)
    - Text label inside segment: "P{number}" in white small monospace RichText (truncate name if segment is narrow)
    - If segment is wide enough (>100px), also show "{name}" below the number
    - Current (in-progress) phase: draw with a 2px bright white/gold border to make it visually pop
    - Selected phase (selected_phase_idx): draw with a 1px contrasting border
    - Use `ui.painter()` to draw filled rects + text directly (more control than egui buttons for this proportional layout)
    - On click detection: use `ui.interact()` on each segment rect — `Response.clicked()` triggers:
      - Set `state.selected_phase_idx = Some(idx)`
      - If phase has commit_range: set `state.git_diff_window = DiffWindow::CommitRange { from, to }` and `state.git_diff_requested = true`
    - Hover tooltip on each segment (use `Response.on_hover_text()`):
      ```
      Phase {number}: {name}
      Goal: {goal}
      Status: {status_label}
      Files: {file_count}
      ```
      This satisfies GSDP-05.

    Place the navigator call in toolbar.rs after `draw_git_diff_controls()`:
    ```rust
    draw_gsd_phase_controls(ui, state);
    ```
    Where `draw_gsd_phase_controls` is a simple wrapper that calls into `draw_panels::draw_gsd_phase_navigator`.

    **2. File Detail Panel Phase Section** — `draw_gsd_phase_detail_section()` in pmat_panel.rs:

    Add after the existing clippy/coverage/rank sections, gated on `state.color_mode == ColorMode::GsdPhase`:
    - CollapsingHeader "Phase History" with id_salt "gsd_phase_detail"
    - If no gsd_phase_report or no selected file: show "No phase data" fallback
    - Look up the selected file in the report:
      - If file is in a phase: show "Phase {number}: {name}" with status color
      - Show the phase goal
      - If the file appears in multiple phases (check all phases, not just by_file which stores most recent):
        show "Also modified in: Phase {X}, Phase {Y}" as weak text
      - If file is NOT in any phase: show "Not referenced in any GSD phase plan. Consider whether this file needs review." in weak amber text — this makes unassociated files actionable

    **3. Color Legend GsdPhase Branch** — in draw_color_legend() in draw_panels.rs:

    When ColorMode::GsdPhase:
    - Green swatch + "Completed"
    - Amber swatch + "In Progress"
    - Blue swatch + "Planned"
    - NO_DATA_GRAY swatch + "Not in any phase"
    - After swatches, add coverage stat: compute (files_in_any_phase / total_files_in_snapshot * 100) and show "{X}% phase coverage" as small weak text
    - Use the existing draw_swatch() helper for consistency

    **4. Hover Tooltip on Treemap Nodes**:

    In the existing tooltip rendering code (find where hovered_path tooltip is drawn — canvas.rs or a panel handler), when ColorMode::GsdPhase is active and the hovered file has a phase association via gsd_phase_report:
    - Append to tooltip: "Phase {number}: {name} ({status_label})"
    - If the tooltip system is in renderer code with only RenderContext access, the gsd_phase_report field provides the data
