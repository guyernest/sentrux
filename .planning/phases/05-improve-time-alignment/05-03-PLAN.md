<task type="auto">
  <name>Task 2: Wire delta computation pipeline and reset button</name>
  <files>sentrux-core/src/app/draw_panels.rs, sentrux-core/src/app/state.rs</files>
  <action>
    In draw_panels.rs, add maybe_spawn_delta_thread(app: &mut SentruxApp):
    - Guard: if !app.state.delta_requested { return; }
    - Clear flag: app.state.delta_requested = false;
    - Guard: if app.state.delta_running { return; }
    - Guard: let root = match app.state.root_path.clone() { Some(r) => r, None => return };
    - Guard: let selection = match &app.state.timeline_selection { Some(s) => s.clone(), None => { state.delta_report = None; return; } };
    - Clone reports: pmat, coverage, clippy from state
    - Set state.delta_running = true
    - Clone msg_tx
    - Spawn thread "delta-compute":
      1. Call load_nearest_snapshot(&root, selection.epoch_start)
      2. If Some(baseline): call compute_delta_report(&baseline, &pmat, &coverage, &clippy), send DeltaReady
      3. If None: send DeltaReady with empty TimelineDeltaReport (no baseline = no arrows, correct behavior per RESEARCH.md pitfall 3)

    Also in the delta thread, trigger a git diff for the selected range:
    - Build a DiffWindow::CommitRange based on the selection. For Phase selection: use phase.commit_range.0 as from, HEAD as to. For Commit selection: use commit.sha as from, HEAD as to.
    - Send this DiffWindow back to the main thread. The simplest approach: set state.git_diff_window to the CommitRange and state.git_diff_requested = true on the MAIN thread before spawning the delta thread (not inside the thread). This way the existing maybe_spawn_git_diff_thread handles the git diff re-trigger.

    Call maybe_spawn_delta_thread(app) alongside the other thread spawners in draw_toolbar_panel.

    In draw_timeline_navigator (from Plan 02), when a timeline selection changes:
    - Set state.delta_requested = true
    - Set state.git_diff_window = DiffWindow::CommitRange { from, to } based on selection
    - Set state.git_diff_requested = true
    - When selection is cleared (deselect), set state.delta_report = None and optionally restore previous git_diff_window

    Add draw_timeline_reset_button(ui: &mut egui::Ui, state: &mut AppState):
    - Only visible when state.timeline_selection.is_some()
    - Small button with "x" or "Reset" label
    - On click: state.timeline_selection = None; state.delta_report = None; state.delta_requested = false
    - Call this in draw_timeline_navigator after the rows, before the commit bar

    In state.rs, ensure clear_stale_state also clears delta_requested.
