    1. Guard: if state.gsd_phase_report.is_none() { return; } -- visible regardless of color_mode per CONTEXT.md decision ("timeline is a navigation tool, not a color overlay").

    2. Guard: if state.gsd_phase_running { ui.label("Loading timeline..."); return; }

    3. Clone report and commits from state to release borrows:
       ```rust
       let report = state.gsd_phase_report.as_ref().unwrap().clone();
       let commits = state.commit_summaries.clone();
       let milestones = state.milestone_infos.clone();
       ```

    4. Get available width: `let total_width = ui.available_width();`

    5. Draw time ticks row (12px height):
       - If commits.is_empty(), skip
       - Compute epoch_min and epoch_max from commits
       - Call choose_tick_granularity_secs(epoch_max - epoch_min) to get tick interval
       - Since commits use equal-width segments (not wall-clock proportional), map tick marks by finding the commit whose epoch is nearest to each tick value and draw at that commit's x position
       - Use ui.painter().text() with small monospace font (9px)
       - Format ticks: for spans < 1 day, show "HH:MM"; for spans < 30 days, show "MMM DD"; for longer, show "YYYY-MM"
       - Use simple epoch arithmetic (no chrono): month names via static array, day/hour/min from epoch % 86400

    6. Draw milestone row (16px) ONLY if milestones.len() > 1:
       - Equal-width segments per milestone
       - Each segment labeled with milestone name
       - Click handling: set state.timeline_selection = Some(TimelineSelection { kind: Milestone, index: i, epoch_start })
       - Hover: show tooltip with milestone name + phase count

    7. Draw phases row (18px height):
       - If timeline_selection selects a milestone, filter to only that milestone's phase_indices; otherwise show all phases
       - Equal-width segments for visible phases
       - Color each segment with gsd_phase_color(phase.status) from colors.rs
       - Label with phase number and short name (truncate if segment < 60px)
       - Click: set state.timeline_selection = Some(TimelineSelection { kind: Phase, index: phase_idx, epoch_start: first commit epoch in that phase })
       - If clicking already-selected phase, deselect (set timeline_selection = None) per CONTEXT.md
       - Hover: tooltip with phase name, goal (first 80 chars), status, file count

    8. Draw commits row (14px height):
       - If timeline_selection selects a phase, filter commits to commits.iter().filter(|c| c.phase_idx == Some(selected_phase_idx)); otherwise show all commits
       - Equal-width segments per visible commit
       - Apply MIN_SEG_WIDTH = 7px overflow handling: if segment_width < 7px and commit count > bar_width/7.0, show first N commits then a "..." overflow segment with tooltip "M more commits"
       - Label: short_sha (only if segment width >= 40px, otherwise no label)
       - Click: set state.timeline_selection = Some(TimelineSelection { kind: Commit, index: commit_idx_in_vec, epoch_start: commit.epoch })
       - If clicking already-selected commit, deselect per CONTEXT.md
       - Hover: tooltip with full message, author, date, file_count, lines changed

    9. After all rows, apply mutations to state:
       - Collect new_selection from interaction responses
       - Apply to state.timeline_selection
       - If selection changed, set state.delta_requested = true (Plan 03 will handle)
