    **In pmat_panel.rs** — extend `draw_pmat_panel()` to show new analysis sections:

    1. After the existing PMAT file detail section (or at the end of `draw_pmat_panel`), add three new sections when a file is selected (`state.selected_path.is_some()`):

    **Code Rank section** (`draw_rank_section`):
    - Check `state.graph_metrics_report.is_some()`.
    - Extract basename from selected path using `.rsplit('/').next()`.
    - Look up in `by_filename`. If found, display:
      - "PageRank: {:.4}" with the pagerank value
      - "Degree Centrality: {:.3}" with degree_centrality
      - "Betweenness: {:.3}" with betweenness_centrality
      - "In-degree: {}" and "Out-degree: {}" with in/out degree
    - If basename matches multiple files (common for mod.rs), show a small note: "(matched by filename)".
    - If not found, show "No centrality data for this file".
    - Use a collapsing header labeled "Code Rank".

    **Coverage section** (`draw_coverage_section`):
    - Check `state.coverage_report.is_some()`.
    - Look up selected path in `by_path`. If found, display:
      - "Lines: {:.1}%" with lines.percent
      - "Functions: {:.1}%" with functions.percent
    - If not found but coverage data exists, show "No coverage data for this file".
    - If coverage_report is None, show "Coverage not collected — click Run Coverage in toolbar".
    - Use a collapsing header labeled "Test Coverage".
    - Format coverage display to accommodate future delta display (e.g., just show "85.0%" now — later "+3%" can be appended).

    **Clippy section** (`draw_clippy_section`):
    - Check `state.clippy_report.is_some()`.
    - Look up selected path in `by_file`. If found, display:
      - "{total} warnings" as header
      - For each category in by_category (sorted by count descending): "{count} {category}" on its own line
    - If not found, show "No clippy warnings" (this is good news).
    - Use a collapsing header labeled "Clippy Analysis".

    **In toolbar.rs** — add "Run Coverage" button and gate Coverage ColorMode:

    1. Find where ColorMode buttons are rendered (likely a loop over `ColorMode::ALL` or individual buttons).
    2. For `ColorMode::Coverage`: if `state.coverage_report.is_none()`, render the button as grayed out / disabled. Add tooltip: "Run coverage first (button below)" or "cargo-llvm-cov not installed — run: cargo install cargo-llvm-cov" if `check_llvm_cov_available()` returns false. Use `OnceLock<bool>` to cache the availability check (same pattern as `check_pmat_available`).
    3. Add a "Run Coverage" button (or icon button) in the toolbar. When clicked:
       - Set `state.coverage_running = true`.
       - Clone `state.root_path` and the scan channel sender (`tx: Sender<ScanMsg>`).
       - Spawn `std::thread::spawn` with a closure that:
         a. Calls `run_coverage(&root, 0)` (gen=0 since coverage is not generation-tracked).
         b. On Some(report): sends `ScanMsg::CoverageReady(report)` via tx.
         c. On None: sends `ScanMsg::CoverageError("cargo-llvm-cov failed".into())` via tx.
       - While `state.coverage_running` is true, show a spinner or "Running..." text on the button.
    4. The toolbar needs access to the `Sender<ScanMsg>` channel. Check how the existing scan trigger works (likely `state` carries a channel clone or the toolbar function receives it as a parameter). Follow the same pattern.

    **Community highlight** — in `state.rs` or wherever file selection is handled:

    1. Find where `state.selected_path` is set (likely in the click handler in `app/mod.rs` or `update_loop.rs`).
    2. When a file is selected AND `state.color_mode` is `ColorMode::Risk`:
       - Access `state.snapshot` import edges.
       - Run BFS from the selected file path over import edges (both directions — from and to) to find the connected component.
       - Store the result in `state.community_highlight = Some(visited_set)`.
    3. When a file is deselected or color mode changes away from Risk, set `state.community_highlight = None`.
    4. The renderer already has spotlight logic using `connected_files` in rects.rs — check if `community_highlight` can be passed through the same mechanism or needs a separate path. If the existing spotlight overlay logic works (it brightens/highlights connected files), reuse it by passing `community_highlight` as the connected_files set.

    **Important implementation notes:**
    - The BFS for community detection uses the snapshot's import graph. Check the actual field name — it may be `snapshot.import_graph`, `snapshot.edges`, or `snapshot.files[].imports`. Read the Snapshot struct to find the correct field.
    - If the import graph is not directly accessible as edge pairs, construct edges from the file-level import lists (each file has a list of files it imports).
    - The coverage button should only be enabled when `state.root_path.is_some()` and `!state.coverage_running` and `!state.scanning`.
