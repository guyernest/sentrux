    - risk_color(Some(0.01), Some(95.0), Some(0)) returns greenish (low centrality + well covered + no lints)
    - risk_color(None, None, None) returns cool/neutral (all defaults)
  </behavior>
  <action>
    **In layout/types.rs:**

    1. Add `Coverage` and `Risk` variants to `ColorMode` enum, positioned AFTER `TdgGrade` and BEFORE `#[serde(other)] Monochrome`. This is CRITICAL — serde(other) must remain on the last variant.

    2. Update `ColorMode::ALL` to include `Coverage` and `Risk`.

    3. Update `label()` match: `Coverage => "Coverage"`, `Risk => "Risk"`.

    **In renderer/colors.rs:**

    1. Add `coverage_color(line_pct: f64) -> Color32`: Map 0-100% to red-yellow-green gradient. pct >= 80 = green, 40-80 = yellow blend, < 40 = red. Use same RGB interpolation pattern as `tdg_grade_color` (t = pct/100, clamp 0..1, r = 30 + (1-t)*225, g = 180*t, b = 40).

    2. Add `risk_color(pagerank: Option<f64>, coverage_pct: Option<f64>, clippy_count: Option<u32>) -> Color32`:
       - pr = pagerank.unwrap_or(0.0).clamp(0.0, 1.0)
       - uncovered = 1.0 - coverage_pct.unwrap_or(50.0).clamp(0.0, 100.0) / 100.0
       - lint_factor = 1.0 + (clippy_count.unwrap_or(0) as f64 + 1.0).ln() / 5.0
       - raw = pr * uncovered * lint_factor
       - Normalize: t = (raw / max_raw).clamp(0.0, 1.0) — but max_raw is per-project. For this function, accept a `max_raw: f64` parameter for project-level normalization. If max_raw <= 0.0, treat as 1.0.
       - Same green-to-red gradient: r = 30 + t*225, g = 180*(1-t), b = 40.
