# Phase 02.1: Rust Deep Analysis - Context

**Gathered:** 2026-03-14
**Status:** Ready for planning

<domain>
## Phase Boundary

Integrate three Rust-specific analysis sources into sentrux — PMAT code rank (PageRank, centrality, community detection), test coverage via cargo-llvm-cov, and clippy advanced linting. Each provides per-file data that feeds into treemap coloring and detail panels. This phase collects and displays current scores; temporal comparison (tracking deltas over time) will compose with the git diff overlay in Phase 3.

</domain>

<decisions>
## Implementation Decisions

### Test Coverage (cargo-llvm-cov)
- Runs **on demand only** — user clicks a "Run Coverage" button in toolbar
- Coverage ColorMode is **disabled/grayed out** in the switcher until coverage data is collected
- Claude's discretion on what data to show per file (line coverage %, branch coverage, or both — based on what cargo-llvm-cov provides)
- Coverage is expected to take 30s-2min depending on project size — show progress indicator

### Clippy Analysis
- Lint groups: **default + pedantic + nursery** (not restriction)
- Display: **count + categories** per file (e.g., "5 warnings: 2 complexity, 2 style, 1 correctness")
- Run timing: Claude's discretion (automatic if fast enough, on-demand if slow)

### PMAT Code Rank
- Color mode: **hot = risky** — combine high PageRank with low coverage or high clippy warnings to highlight danger zones
- Community detection: **click a file to highlight its community members** on the treemap (not a global ColorMode, an interaction mode)
- File detail panel shows: all available metrics — PageRank score with label, centrality measures (degree + betweenness), community members list
- No standalone "PageRank" ColorMode — the risk-combined view is more actionable

### Tool Requirements
- All tools required: PMAT + cargo-llvm-cov + clippy
- Missing tools: grayed-out features with tooltip showing install command (same pattern as PMAT but non-blocking for scan — only PMAT blocks scan, coverage/clippy features gracefully degrade)

### Design Principle: Relative Scores
- The **trend/delta of scores is as important as the absolute score**
- Phase 2.1 collects current scores; Phase 3 (git diff overlay) will show how metrics changed over time windows
- File detail panel should show data in a way that future delta display can be added (e.g., "Coverage: 85%" today → "Coverage: 85% (+3%)" later)
- The goal is to focus developer effort on areas that need improvement, not just show static numbers

### Claude's Discretion
- cargo-llvm-cov output format selection (JSON, lcov, or summary)
- Clippy run timing (auto vs on-demand)
- Risk color formula (how to combine PageRank + coverage + clippy into a single risk signal)
- Progress indicator design for coverage collection
- Whether clippy needs a separate ColorMode or feeds into the risk view

</decisions>

<specifics>
## Specific Ideas

- "The relative score is as important as the score itself" — user wants to track improvement/degradation over time
- "The goal is to allow improvement in quality and focusing efforts to areas that need these improvements" — this is a quality improvement tool, not just a dashboard
- Clicking a file to highlight its community is a novel interaction — makes architectural clusters discoverable without a separate view

</specifics>

## Existing Code Insights

### Reusable Assets
- `analysis/pmat_adapter.rs:run_pmat_command<T>` — generic subprocess runner, reuse for `pmat analyze graph-metrics` and `cargo clippy`/`cargo llvm-cov`
- `core/pmat_types.rs` — pattern for defining subprocess output types with serde
- `app/panels/pmat_panel.rs` — file detail panel infrastructure, add coverage/clippy/rank sections
- `renderer/colors.rs:tdg_grade_color` — gradient color mapping pattern, reuse for coverage and risk gradients
- `layout/types.rs:ColorMode` — enum with 5 variants, add Coverage and Risk modes

### Established Patterns
- Subprocess → parse JSON → store on `ScanReports` → `AppState` → `RenderContext` → renderer (Phase 2 pipeline)
- ColorMode dispatch in `renderer/rects.rs:file_color()` — add new match arms
- `OnceLock` caching for tool availability checks (`pmat_adapter.rs`)

### Integration Points
- `ScanReports` — add `coverage: Option<CoverageReport>`, `clippy: Option<ClippyReport>`, `code_rank: Option<CodeRankReport>`
- `AppState` — store reports for panel access
- `RenderContext` — pass reports to renderer for color dispatch
- Toolbar — "Run Coverage" button, new ColorMode entries (Coverage, Risk)

</code_context>
