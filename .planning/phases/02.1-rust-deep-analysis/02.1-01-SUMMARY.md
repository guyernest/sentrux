## Decisions Made

- risk_color() accepts a `max_raw: f64` parameter for project-level normalization instead of a hardcoded constant (empirical max varies by project size and PageRank distribution)
- coverage_color() uses the same r/g/b interpolation as tdg_grade_color for visual consistency: `t = pct/100`, `r = 30 + (1-t)*225`, `g = 180*t`, `b = 40`
- ColorMode::Coverage and Risk fall back to monochrome in file_color() — wiring to actual per-file data happens in 02.1-02 after ScanReports is extended
- CoverageReport::from_output takes scan_root and normalizes with trailing slash to avoid partial path prefix matches
