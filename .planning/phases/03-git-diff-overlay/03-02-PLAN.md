  <action>
    1. In `sentrux-core/src/app/state.rs`, add fields to AppState (in the "Analysis reports" section, after coverage fields):
       - `pub git_diff_report: Option<GitDiffReport>` — None until computed
       - `pub git_diff_running: bool` — true while background thread active
       - `pub git_diff_window: DiffWindow` — active window selection
       - `pub git_diff_requested: bool` — flag set by toolbar
       - `pub git_diff_custom_n: u32` — custom commit count input
       Add imports for `GitDiffReport` from `crate::core::pmat_types` and `DiffWindow` from `crate::metrics::evo::git_walker`.
       Set defaults in `Default` impl: git_diff_report: None, git_diff_running: false, git_diff_window: DiffWindow::default(), git_diff_requested: false, git_diff_custom_n: 10.

    2. In `sentrux-core/src/app/scanning.rs`:
       - Add `ScanMsg::GitDiffReady(report)` handler (follow CoverageReady pattern exactly):
         ```
         state.git_diff_report = Some(report);
         state.git_diff_running = false;
         ctx.request_repaint();
         ```
       - Add `ScanMsg::GitDiffError(msg)` handler:
         ```
         eprintln!("[git_diff] {msg}");
         state.git_diff_running = false;
         ctx.request_repaint();
         ```
       - In the scan-reset path (where coverage_report is set to None on new scan), also reset:
         `git_diff_report = None`, `git_diff_running = false`
         Do NOT reset git_diff_window or git_diff_custom_n (user selections persist).

    3. In `sentrux-core/src/app/draw_panels.rs`:
       - After the `coverage_requested` handler block, add git_diff_requested handler (same pattern):
         ```rust
         if app.state.git_diff_requested {
             app.state.git_diff_requested = false;
             maybe_spawn_git_diff_thread(app);
         }
         ```
       - Implement `fn maybe_spawn_git_diff_thread(app: &mut SentruxApp)`:
         - Guard: return if root_path is None, or git_diff_running, or scanning
         - Set `app.state.git_diff_running = true`
         - Clone `app.scan_msg_tx`, `root`, and `app.state.git_diff_window`
         - Call `crate::analysis::git_diff_adapter::spawn_git_diff_thread(root, window, msg_tx)`
         - On thread spawn failure, set `git_diff_running = false` and log error

    4. In `sentrux-core/src/renderer/mod.rs`, add to RenderContext:
       - `pub git_diff_report: Option<&'a crate::core::pmat_types::GitDiffReport>`
       - Initialize it in the RenderContext builder (wherever RenderContext is constructed — likely in canvas.rs or the main render function) from `state.git_diff_report.as_ref()`

    5. In `sentrux-core/src/renderer/rects.rs`, update `color_by_git_diff`:
       - Change from fallback-only to using `ctx.git_diff_report` (the RenderContext field added above)
       - The logic should match what Plan 01 established: None -> theme_config.file_surface, path not found -> muted gray, is_new_file -> new file color, otherwise -> intensity gradient

    6. Find where RenderContext is constructed (likely in `sentrux-core/src/app/canvas.rs` or similar) and add `git_diff_report: state.git_diff_report.as_ref()` to the constructor.
