  <action>
    1. In `sentrux-core/src/app/toolbar.rs`, add `draw_git_diff_controls()`:
       - Only render when `state.color_mode == ColorMode::GitDiff` (per CONTEXT.md: "Window selector only visible when GitDiff mode active")
       - Show a horizontal row with separator and "Window:" label
       - Iterate `DiffWindow::PRESETS`, render each as `selectable_label(selected, label)` where selected = `state.git_diff_window == window`
       - On click: set `state.git_diff_window = window`, set `state.git_diff_requested = true`
       - After presets, show custom commit count input:
         - "N:" label + `egui::DragValue::new(&mut state.git_diff_custom_n).range(1..=999).speed(1.0)`
         - "go" button next to it. On click OR Enter key: set `state.git_diff_window = DiffWindow::CommitCount(state.git_diff_custom_n)`, set `state.git_diff_requested = true`
       - If `state.git_diff_running`, show a "computing..." indicator (disabled text or spinner)
       - Call this function from the main toolbar draw function, after the color mode buttons

    2. Add auto-trigger behavior:
       - In the color mode selector (where ColorMode buttons are drawn), detect when user switches TO GitDiff mode
       - If `state.git_diff_report.is_none()` and `!state.git_diff_running`, auto-set `state.git_diff_requested = true` to trigger computation with the current window
       - This ensures the user sees data immediately when switching to GitDiff mode without needing to click a window preset first

    3. In `sentrux-core/src/app/draw_panels.rs` (or a new small module), add `draw_color_legend()`:
       - Called from the main UI layout, below the toolbar (or at the bottom of the toolbar area)
       - Dispatches by `state.color_mode`:
         - `ColorMode::GitDiff` -> draw_git_diff_legend():
           - Horizontal row with small text
           - Muted gray swatch (14x10 rect) + "unchanged" label
           - Gradient strip (80px wide, 12 color steps from git_diff_intensity_color(0) to git_diff_intensity_color(1)) + "few -> many changes" label
           - Teal swatch (14x10 rect using git_diff_new_file_color()) + "new file" label
           - If no report: "(no data -- select a window)" hint
         - `ColorMode::TdgGrade` -> draw_tdg_legend():
           - Show grade badges: A+ (green) through F (red) as small colored squares with labels
         - `ColorMode::Coverage` -> draw_coverage_legend():
           - Green (100%) to red (0%) gradient strip + "well-covered -> uncovered" label
           - Muted gray swatch + "no data" label
         - `ColorMode::Risk` -> draw_risk_legend():
           - Green (safe) to red (risky) gradient strip + "safe -> risky" label
         - Other modes (Language, Heat, Git, Monochrome): no legend (return early)
       - Use `egui::painter().rect_filled()` for color swatches
       - Use `egui::RichText::new(...).small().weak()` for labels to keep legend unobtrusive

    4. Wire `draw_color_legend()` into the main UI layout:
       - Find where the toolbar is rendered (likely in `draw_panels.rs` or `canvas.rs`)
       - Add `draw_color_legend(ui, &state)` call after the toolbar rendering
       - Ensure it appears in a horizontal strip below the toolbar

    5. Wire `draw_git_diff_controls()` into the toolbar rendering.
