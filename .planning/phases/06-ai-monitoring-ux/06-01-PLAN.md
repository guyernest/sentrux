<objective>
Add TDG grade complexity_penalty to the risk formula so that simple hub files (e.g. mod.rs with A+) no longer false-alarm as high-risk.

Purpose: mod.rs and similar architectural hub files have high PageRank but trivially simple code. The old formula gives them false high-risk scores. Multiplying by `1.0 - grade_to_t(grade)` makes A+ files near-zero risk and F-grade files retain their full risk weight.

Output: Updated `compute_raw_risk()` with `complexity_penalty: f64` parameter, updated call sites in `color_by_risk()` and `compute_max_risk_raw()`, and new unit tests covering the A+ near-zero and F full-penalty cases.
</objective>

```rust
pub fn compute_raw_risk(pagerank: f64, coverage_pct: f64, clippy_count: u32) -> f64 {
    let pr = pagerank.clamp(0.0, 1.0);
    let uncovered = 1.0 - coverage_pct.clamp(0.0, 100.0) / 100.0;
    let lint_factor = 1.0 + (clippy_count as f64 + 1.0).ln() / 5.0;
    pr * uncovered * lint_factor
}
```

fn color_by_risk(ctx: &RenderContext, path: &str) -> Color32 {
    let max_raw = ctx.max_risk_raw;
    let basename = path.rsplit('/').next().unwrap_or(path);
    let pagerank = ctx.graph_metrics_report
        .and_then(|gm| gm.by_filename.get(basename).map(|&idx| gm.data.nodes[idx].pagerank))

    let coverage_pct = ctx.coverage_report
        .and_then(|cov| cov.by_path.get(path).map(|&idx| cov.files[idx].summary.lines.percent));
    let clippy_count = ctx.clippy_report
        .and_then(|r| r.by_file.get(path))
        .map(|d| d.total);
    colors::risk_color(Some(pagerank), coverage_pct, clippy_count, max_raw)
}
